CT_Mediastreamer_01

Mediastreamer – Requesting Live tuning playback via webservice interface

Positive

 XG1

1

1.Mediastreamer executable should be running

Webservice Interface

string-ocapId

1.TM loads Mediastreamer_agent via the test agent.

2.TM gets an ocapid from the streaming details page of the FW and sends it to Mediastreamer_agent to generate request url.

3.Mediastreamer_agent will get RecorderId from wbdevice.dat file in XG1 and frames the query url for live tune like  “http://localhostip:port/videoStreamInit?recorderId=&live=ocapid” and send to the mediastreamer.

4.Upon receiving the Json response from mediastreamer, Mediastreamer_agent will extract the Json parameters like error code and error description and send to TM.

5.TM will do the error checking by verifying Error code and Error description parameters.

Checkpoint 1 Error code and Error description parameter of Json response is verified as success or failure.                                            

High

CT_RMFMediaStreamer_12

Recorder-To check scheduling current recording.

Positive

XG1

 

1.RMFMediastreamer executable should be running.

2.Device time should be in current time of UTC.

3. Two files should be created in the name of “stt_received” and “stage4” in “tmp” path of device.

4. In rmfconfig.ini file the parameter “FEATURE.LONGPOLL.URL” should be pointing to TM.

Json Interface

Json Interface- source id, duration recording_id, start_time.

1.TM loads Mediastreamer_agent via the test agent.

2.TM gets an source_id from the streaming details page of the FW and sends it to Mediastreamer_agent to generate request url.

3.TM pass the parameters like duration and recording_id to Mediastreamer_agent.

3.Mediastreamer_agent will frame the json message to schedule the recording and send to TDK_Recorder_server which is present in TM.

4.Status of the Json response from Mediastreamer to TDK_Recorder_Server getting extracted by TM.

5 Mediastreamer_agent  will do the error checking by verifying ocapri_logs.

6.Depends on the result of above step Mediastreamer_agent sends SUCCESS or FAILURE to TM.

Checkpoint 1 Status from the TDK_Recorder_server.

Checkpoint 2 Verifying the ocapRI_log to check the state of Recording.                                    

High

Mediastreamer_agent

1.TestMgr_MediaStreamer_ScheduleRecording

2.TestMgr_MediaStreamer_checkRecording_status
